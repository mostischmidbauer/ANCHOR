How this model works (summary)

  • If a wallet is connected, identity = did:ethr:<wallet>.
  • If no wallet is available, identity = did:anon:<deviceId>.
  • Each browser has a stable, pseudonymous deviceId stored locally (no server).
  • On “Run Liveness Check”, this page simulates a liveness result and issues a short-lived JWT-shaped token.
  • Your production system can replace the simulation with a real rPPG / anti-deepfake pipeline and move signing into a secure backend.
