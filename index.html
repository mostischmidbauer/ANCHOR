How to use this tool

  • Use it to define a standard Anti-Deepfake Session Token schema for your backend.
  • Bind each session to a stable deviceId and optional wallet-based DID.
  • Use the generated tokens in:
    • API specs / OpenAPI examples,
    • technical documentation for regulators,
    • mock integration tests with your partners.
  • Signature is a fixed local-demo-signature so you can verify tokens with your static verifier page. In production, replace with a server-side key.
